The maximal list is bigger than all the lists in your chain, but it still doesn't contain all real numbers. So you can find an element not in the maximal list (which wasn't in any of the lists in the chain), then another element not in *that* list, and can even go on to make *another* maximal list. But that still won't be all the real numbers, and you can find yet more missing elements...

Your partial order ℙ is (something like) countable subsets of ℝ, ordered by inclusion. You are right that, if ℙ has a maximal element, then ℝ is countable.

So does ℙ have a maximal element? Well, by Zorn's Lemma it would ... if every chain in ℙ had an upper bound. Here you write

Any chain has an upper bound which is the union of all lists in that chain.

But this isn't true! The union of a chain in ℙ is a set of reals, but not necessarily a countable set of reals - and ℙ only contains countable sets of reals. So the union of a chain in ℙ need not be an element of ℙ.

The union of a countable chain in ℙ will indeed be an element of ℙ - but not every chain in ℙ is countable, and Zorn's Lemma needs every chain to have an upper bound in the poset.

The problem is that we are trying to prove that the real numbers are uncountable. So we can't use the fact that they are already uncountable, because the number of elements in any chain can't exceed the number of real numbers. So at the moment we don't know if chain can contain uncountably many elements or not.
Can we exhibit such a chain whose union is uncountable explicitly? We can make one from a well-ordering of ℝ, but that's not very explicit.
