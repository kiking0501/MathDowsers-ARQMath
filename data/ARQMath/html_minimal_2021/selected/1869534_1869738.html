Improved brute force: you may compute the number of divisors of every number in the range [100,70000] whose prime divisors belong to the set {2,3,5,7,11,13}: that requires at most one second of CPU time.
Note that if a number has prime decomposition p1a1⁢p2a2⁢…⁢pkak, the number of divisors is the number of integer vectors (b1,b2,b3,…,bk) such that 0≤bi≤ai. So, you have a1⁢a2⁢…⁢ak divisors for that number.
