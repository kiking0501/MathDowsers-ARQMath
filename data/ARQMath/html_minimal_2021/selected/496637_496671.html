We can use general techniques to solve this Diophantine equation, but the numbers are so small that it does not seem worthwhile. Note that when q=2, then 133-8⁢q is a multiple of 3, and we get q=2, d=39, and therefore p=59.

Now we have found one solution. We need others.

Go back to the equation 8⁢q+3⁢d=133. We have found one solution, q=2, d=39. For any integer k, we therefore have 8⁢(2+3⁢k)+3⁢(39-8⁢k)=133. (Note the cancellation.)

Substitute various values of k. With k=1, for example, we get the solution q=5, d=31. That gives p=64.

Continue, using k=2, 3, and so on. For the sake of reality, we will have to make sure that our values of q, d, p are all ≥0. We cannot use k⁢\gt⁢4, for that would give negative d.

Thank you, "typos," fixed. Any basic number theory book has the material. Given a linear Diophantine equation a⁢x+b⁢y=c, if gcd⁡(a,b) does not divide c, there is no solution, and if it does divide, there is a solution. Divide through by gcd⁡(a,b). So we may assume gcd⁡(a,b)=1 to begin with. Now find one **particular** solution (x0,y0). For small numbers, this is done by inspection. For big numbers, use the **Extended Euclidean Algorithm** to solve a⁢s+b⁢t=1, and multiply s, t by c. (Cont)
(Cont) There is a general theorem that if gcd⁡(a,b)=1 and (x0,y0) is a particular solution, then **all* solutions are given by x=x0+k⁢b, y=y0-k⁢a.
