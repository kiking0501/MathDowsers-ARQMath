The "without-test-function" way of computing dp⁢f=(f∗)p comes another version of what is meant by a tangent vector. That is, there are two (equivalent) definitions of tangent vector.

The first definition of a tangent vector is that a tangent vector v∈Tp⁢M is an equivalence class of derivative of a curve. That is, there is a curve γ:(-ϵ,ϵ)→M for which γ⁢(0)=p, γ′⁢(0)=v. Further, two such curves are called equivalent if in local coordinates, they have the same derivative at p.

The second definition of a tangent vector is that v∈Tp⁢M means that v:C∞⁢(M)→ℝ acts on C∞⁢(M) as a linear transformation satisfying the Leibniz rule: v⁢(f⁢g)=f⁢(p)⁢v⁢(g)+v⁢(f)⁢g⁢(p). That is, v is a derivation of C∞⁢(M).

The link between the two: given any vector v∈Tp⁢M, write it as γ′⁢(0). Then γ′⁢(0) acts on f∈C∞⁢(M) by γ′⁢(0)⁢(f):=dd⁢t|t=0⁢f⁢(γ⁢(t)). Further, it's a not-too-hard-but-not-trivial exercise to show that every derivation arises from this construction.

How does pushforward work in definition one? Well, given v∈Tp⁢M with associated curve γ, and given f:M→N smooth, we create an element f∗⁢(v)∈Tf(p⁢(N) as follows: f∘γ is a curve in N, so (f∘γ)′⁢(0)∈Tf⁢(p)⁢N. Note that no test function is used here.

How does pushforward work in definition two? Well, given v∈Tp⁢M, and smooth f:M→N, we pick a test function g:N→ℝ. Then f∗⁢(v)⁢(g)=v⁢(g∘f).
