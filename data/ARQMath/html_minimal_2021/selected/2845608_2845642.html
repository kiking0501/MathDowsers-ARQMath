It is unfortunate that four answers were posted before anyone pointed out that there is no such thing as the event A∣B. Now my answer does that. This seems to be a confusion that many beginners get into.

Consider a sample space Ω={1,2,3} corresponding to the result of tossing a (fictitious) three-sided die. As our prototypical examples of events, we will take A={1,2} (rolling the number one or two) and B={1,3} (rolling the number one or three).

More generally, an event on Ω is any one of the following: ∅,{1},{2},{3},{1,2},{1,3},{2,3},{1,2,3} (if you are familiar with it, this is nothing than the powerset of Ω).

When we intersect two events, we obtain another event corresponding to both events occurring. In our case, A∩B={1} corresponds to rolling the number one.

A probability measure ℙ is responsible for assigning probabilities to particular events. If our three-sided die is "fair", then ℙ⁢(A)=2/3, ℙ⁢(B)=2/3, and ℙ⁢(A∩B)=1/3.

Technically speaking, A∣B is not an event. When we write A∣B, we are alluding to definition of conditional probability: ℙ⁢(A∣B)=ℙ⁢(A∩B)ℙ⁢(B) (note that the above is only well-defined if ℙ⁢(B)>0). This should be interpreted as the probability that event A occurs given that event B has occurred. In our running example, ℙ⁢(A∣B)=1/32/3=12.
