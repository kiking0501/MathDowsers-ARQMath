(N=5): {5, 12, 7, 18, 8, 6, 5, ...}

Your conjecture is true, and it can be proven with a little number theory and some of the computation that you've already done.

The function that you use in the composite-N case, the sum of all the prime divisors (with multiplicity) of N, is sometimes known as the integer log or the potency of N; you can find more about it at https://oeis.org/A001414 . For convenience's sake, I'm going to write this as logℕ⁡(n) going forwards.

We're going to need to prove a few basic properties of logℕ:

  • For all n, logℕ⁡(n)≤n. This can be proven by (strong) induction: if n is prime, then obviously logℕ⁡(n)=n≤n. Otherwise, let p be the smallest prime factor of n; then np≥p, and so logℕ⁡(n)=p+logℕ⁡(np) ≤p+np (by the induction step) ≤np+np (since p≤np) =2p⁢n ≤n (since p≥2).
  • For all composite n⁢\gt⁢4, logℕ⁡(n)⁢\lt⁢n (i.e., the inequality is strict). The argument is similar: let p be the smallest prime divisor of n and note that logℕ⁡(n)≤p+np. But now, if p=2 then the right hand side is 2+n2, and 2+n2⁢\lt⁢n for n⁢\gt⁢4, so logℕ⁡(n)⁢\lt⁢n; otherwise (i.e., if p⁢\gt⁢2) we have logℕ⁡(n)≤p+np≤np+np (as above) =2p⁢n \lt⁢n (since p⁢\gt⁢2).
  • Finally, one further refinement: for all composite n⁢\gt⁢45, not of the form n=2⁢p for prime p, we have logℕ⁡(n)⁢\lt⁢25⁢n. We argue as before: first, we can find a divisor d of n with 3≤d≤nd. (If 4|n then take d=4; otherwise, let d be the smallest odd prime divisor of n.) But now, if d=3 then logℕ⁡(n)≤3+n3 \lt⁢n15+n3 (since n⁢\gt⁢45) =25⁢n. If d=4 then logℕ⁡(n)≤4+n4 \lt⁢320⁢n+n4 (since n⁢\gt⁢27) =25⁢n. If d=5 then logℕ⁡(n)≤5+n5 \lt⁢n5+n5 (since n⁢\gt⁢25) =25⁢n. And finally, if d⁢\gt⁢5 then logℕ⁡(n)≤d+nd≤nd+nd=2d⁢n⁢\lt⁢25⁢n.

Now, we can show that for all N⁢\gt⁢45, we'll have either S⁢(N)⁢\lt⁢N or S⁢(S⁢(N))⁢\lt⁢N. Since this process can be repeated until we get to N≤45 and we already know that all N≤45 (except for 4) go into the 5-cycle, then this implies that all N will fall into this cycle. This breaks down into two cases: if N is composite, then we know that S⁢(N)=logℕ⁡(N)⁢\lt⁢N by the second basic property of logℕ. Finally, we're left with the case that N=p, a prime.

To handle the prime case, note that by simple extensions of Bertrand's Postulate we have that for any constant ϵ there's an n0 such that for all p⁢\gt⁢n0 the prime after p will always be \lt⁢(1+ϵ)⁢p; this means that for all sufficiently large p, S⁢(p)⁢\lt⁢(2+ϵ)⁢p. In particular, it's known that for all p⁢\gt⁢25, pnext⁢(p)⁢\lt⁢(1+15)⁢p, so S⁢(p)=p+pnext⁢(p)⁢\lt⁢115⁢p.

But S⁢(p) will always be composite (it's divisible by 2, after all!) and it can never be of the form 2⁢q for prime q (can you see why?) so for all p⁢\gt⁢25 we have S⁢(S⁢(p))⁢\lt⁢25⁢S⁢(p) (by the third basic property of logℕ, since S⁢(p) is large enough for that to hold) \lt⁢25⋅115⁢p (by the extended Bertrand's postulate) =2225⁢p⁢\lt⁢p.

Now, knowing that for N⁢\gt⁢45 we always have either S⁢(N)⁢\lt⁢N (if N is composite) or S⁢(S⁢(N))⁢\lt⁢N (if N is prime), we can 'hopscotch' our way downwards to the range that your computer simulations have covered, by taking one or two steps along the path as appropriate. For instance, suppose N=199. Since N is prime, we take a 'hop' of two steps: S⁢(S⁢(199))=S⁢(199+211)=S⁢(410)=2+5+41=48. Now 48 is composite, so we take a single step: S⁢(48)=2+2+2+2+3=11. And now this is inside our 'range', so we know it falls into the loop. This same procedure will work for any N⁢\gt⁢45, eventually carrying us to a number less than 45 where we're assured of falling into the 5-cycle.
