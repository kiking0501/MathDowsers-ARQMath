Associativity of logical connectives

According to the precedence of logical connectives, operator → gets higher precedence than ↔ operator. But what about associativity of → operator?

The implies operator (→) does not have the associative property. That means that (p→q)→r is not equivalent to p→(q→r). Because of that, the question comes op how p→q→r should be interpreted.

The proposition p→q→r can be defined in multiple ways that make sense:

  • (p→q)→r (left associativity)
  • p→(q→r) (right associativity)
  • (p→q)∧(q→r)

Which one of these definitions is used?

@Ovi: some authors use → for the logical connective (which could appear in a formula) and ⇒ for logical implication at the meta level. Of course, in first-order logic, for sentences ϕ and ψ, we have ϕ⊧ψ if and only if ϕ⊢ψ if and only if ⊢ϕ→ψ, so in that notation ϕ⇒ψ is equivalent to ϕ→ψ being logically valid.

I think (p→q)∧(q→r) is the most useful interpretation of p→q→r. It corresponds to natural language when we say something like "A implies B, which implies C".

A related convention is x⁢\lt⁢y⁢\lt⁢z meaning x⁢\lt⁢y∧y⁢\lt⁢z. The same three cases arise when \lt is treated as a function yielding a boolean integer like in many programming languages.

The most common convention I have seen is that p→q→r means p→(q→r) which is equivalent to (p∧q)→r. This convention is very common in type theory, because it works well with the Curry-Howard isomorphism. Under this interpretation, a→b is read as "we haveafunction from a to b" and p→q→r means "we have a function that, given p, constructs a function from q to r.".
