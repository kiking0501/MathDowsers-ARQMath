It is called the Cartesian product!

I think, more generally, you need guidance on the notation f:A×B→C, which is the notation for "a function called f, from the Cartesian product A×B of sets A and B, to the set C" (the Cartesian product A×B={(a,b):a∈A,b∈B} is the set of all ordered pairs with things taken from A and B). More generally, the format is

function name:domain→codomain

But this notation f:A×B→C often says nothing about what the function actually does to pairs (a,b)∈A×B to produce some f⁢(a,b)∈C, unless f happens to have a particularly descriptive name/symbol. In this case, you'll often see functions introduced in "two parts",

\begin{align*} f \colon A \times B &\to C \\ (a, b) &\mapsto \text{however a,b determine f⁢(a,b)} \end{align*}

where the first line specifies the function name and all the sets we need, and the second line actually tells us what f does to the pairs (a,b) (and note the new symbol ↦, which is used like → above. But → is used with sets, the domain and codomain, while ↦ is used between the actual input and output, to explain what happens to elements in the sets).

So you'll never see notation like ℝm⋅ℝn→ℝm+n, with the function placed between sets. Instead, you'll see the function name/notation in the place of f, put before the domain. So things like

⋅:ℝn×ℝn→ℝ is a (mildly confusing) notation saying that there's a function called "⋅" that takes two vectors in ℝn, and gives you back a real number (we can assume it's the standard dot product).

×:ℝ3×ℝ3→ℝ3 would be the (somehow more confusing) notation to say there's a function called "×" that takes two vectors in ℝ3 and returns another vector in ℝ3; probably it's the standard cross product on ℝ3.

For a slightly-less-weird-looking example, we might use

+:ℝn×ℝn→ℝn to say that we have an operation called "+" that takes pairs of vectors in ℝn, and returns a single vector in ℝn (and unless it's stated otherwise, everyone would assume "+" means exactly what you think it means).

The domain and codomain can come in all sorts of varieties. For example, functions don't have to be defined on pairs of things, in which case our domain isn't going to be a Cartesian product. So to talk about the standard square root function, we might write

:ℝ≥0→ℝ≥0.

Or maybe we're handed a function with a fairly cryptic name,

\begin{align*} \operatorname{ev} \colon M_{n \times n}(\Bbb R) \times \Bbb R^n &\to \Bbb R^n \\ (A, \vec{v}) &\mapsto A\vec{v} \end{align*}

but with practice, we can see it's the evaluation map that takes an n×n matrix with real entries and a vector in ℝn, and applies A to v→.

Note that many of the examples had the format ℝn×ℝn→ℝn; only the prefix, the function name before :, could help us tell the functions apart. The part of the notation after the : is really only good for domain and codomain, not the function itself (e.g., vector addition versus cross product).
