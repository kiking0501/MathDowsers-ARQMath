The correct approach is: n≤n for n whatever; thus, ≤ is *reflexive*.
≤ is **not** *symmetric*, because 3≤4 but not 4≤3 (a counterexample is enough to disprove).
≤ is *transitive* because if m≤n and n≤p then m≤p, for n,m,p whatever.

I've already answered in the comemnt above; with a *conditional* definition, if the antecedent is not satisfied, the falsity of the consequent does **not** count as a counter-example. Thus, with x-y≠0 we have to check that **every time** that it holds, i.e. n-m≠0, then also m-n≠0 holds. And this is true. The fact that e.g. 3-3=0 (i.e. 3-3≠0 is *false*) means simply that we have to skip it, because it does not count for checking for simmetry.

A relation ∼ on a set A is defined as ∼=(A,ℛ) where ℛ⊆A×A.

We say ∼ is reflexive if for all a∈A, we have a∼a. So you can't wheck reflexivity by choosing (a<-cd,-cd<b) such that a≠b.

We say ∼ is symmetric, for all (a<-cd,-cd<b), if a∼b then b∼a.

And we say ∼ is transitive, for all a,b,c∈A, if a∼b and b∼c then a∼c. So you can't check transitivity with an example like a⁢\nsim⁢b and b⁢\nsim⁢c. It is contrary to the definition.

In order to be more simple, you can consider ∼ as a "true/false" statement. So in your example, is 9≤8 true or false? Is 8≤7 true or false? You need to find two true statements first in order to check transitivity because of its definition. And you can't also choose these statements arbitrarily like 1≤2 and 3≤4 by the definition as well.

Here, I generalize the relation but you can just put ≤ instead of ∼, it won't really change anything since the definitions are general. In addition to that, we call ≤ a partial ordering since it is reflexive, anti-symmetric and transitive.
