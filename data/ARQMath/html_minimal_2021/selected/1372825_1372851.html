The simplest methods you can think of like I≈xmaxn⁢∑i=1nf⁢(xmax⁢in) is usually more than good enough. For example xmax=50 and n=1000 gives the result to 0.2% accuracy.

Option 1:

Cut off the domain on [0,M], for some M chosen such that the integral over [M,∞) is at most half of the desired tolerance. Then use a standard quadrature routine like the trapezoidal rule or Simpson rule on [0,M], with a small enough step size that the error on [0,M] is at most half of the desired tolerance.

Option 2:

Use option 1, but with larger step sizes where the integrand is smaller. This is helpful in your problem, because your integrand is extremely small outside a moderately large interval around 0 (like [0,100]).

Option 3:

Define F⁢(t) to be the even extension of sin⁡(π⁢t) from [0,∞) to ℝ. (So F⁢(t)=sin⁡(π⁢t) for t≥0 and F⁢(t)=sin⁡(-π⁢t) for t<0.)

Write I=∫0∞e-t2/100⁢sin⁡(π⁢t)⁢𝑑t=12⁢∫-∞∞e-t2/100⁢F⁢(t)⁢𝑑t=5⁢∫-∞∞e-s2⁢F⁢(10⁢s)⁢𝑑s

and then perform Gauss-Hermite quadrature.

Option 4:

Similar to option 3, write

I=502⁢∫-∞∞e-s2/2⁢F⁢(50⁢s)⁢𝑑s

and then recognize this as 502⁢E⁢(F⁢(50⁢X)), where X is a standard normal random variable. Then compute this expectation using a Monte Carlo procedure. (This is probably the worst of the four options I've given here.)

Option 5 (perhaps the best, suggested by @uranix):

Write

I=10⁢∫0∞e-s2⁢F⁢(10⁢s)⁢𝑑s

and then use Gauss quadrature directly. As far as I know, this particular form of Gauss quadrature does not have a special name and its parameters are not well-tabulated, but one can use the Golub-Welsch algorithm to generate the nodes and weights directly. The only thing the Golub-Welsch algorithm needs for this case that it can't generate by itself is ∫0∞e-x2⁢𝑑x which is of course π2.

Your F⁢(x) function has a discontinuous derivative, I expect convergence issues with Gauss-Hermite rule. For oscillatory integral Filon rule might be used instead of trapezoidal or Simpson.
@uranix Gauss quadrature is convergent for any bounded, continuous integrand (the weight function not included in the "integrand"). The only question is how high of a degree of a polynomial is required in order to achieve a given estimate (in an appropriate metric depending on the weight function) of the integrand by a polynomial. I don't think that is actually so high in this case. (For instance, you can approximate F by a smooth cutoff of F, which is equal to F outside (-2⁢δ,2⁢δ) but is zero inside (-δ,δ), then apply the standard error estimate to the cutoff.)
