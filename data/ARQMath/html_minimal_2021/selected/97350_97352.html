How does backwards induction work to prove a property for all naturals?

I was reading a blogpost here: http://mzargar.wordpress.com/2009/07/19/cauchys-method-of-induction/

One thing that threw me off was that after the first four large displayed equations, there is the statement "it is sufficient to prove that if the theorem holds for n=m+1, then it holds for n=m."

How is this type of induction valid? I browsed around for things like backward induction, reverse induction, and Cauchy induction, but couldn't find a justification for how this is valid.

With the usual forward induction of verifying a base case and proving P⁢(n)⟹P⁢(n+1), it's easy to intuitively understand how induction will show a property holds for all natural numbers (or at least starting at the base case). But with this reverse induction, it seems to me that if you prove P⁢(m+1)⟹P⁢(m), then if you were able to verify a specific case like P⁢(15), then you would only know it's true for numbers up to 15. How does it actually prove the property for all naturals?

You left out the first part of the sentence! "***Now that we have the case whennis a power of 2 settled,*** _it is sufficient to prove...._" The short answer is that you ***also*** prove that for any n, you can find N >= n for which it is true, usually by proving it true for some infinite set.

You seem to be confused with the method of infinite descent which is used to prove that something is FALSE : If P⁢(0) is false and if P⁢(n) implies that there exists a m<n such that P⁢(m), then P⁢(n) is false for all n. This correspond to prove the anteposition of the standard recurrence principle. (The anteposition of an implication A⇒B is not⁢B⇒not⁢A.)

But that's not the situation in the blogpost. The claim is that P⁢(m+1)⟹P⁢(m) is sufficient to show an inequality holds for all n. There is no question of proving something false.
