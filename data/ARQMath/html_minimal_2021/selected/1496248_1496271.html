Best known upper and lower bounds for ω1

What are the best known lower and upper bounds for ω1? Are there sharper lower bounds than ε0? And are there any known upper bounds which can be explicitly constructed like ε0?

Since ε0 is countable, ε0 must be less than ω1. But in all of the 45 books on logic and set theory on my bookshelf, I can find no bounds better than this. I've been trying to find the answer to this for many years, and it's very frustrating that only one of the infinite cardinals has an explicit construction. The uncountable cardinals seem to be only known to exist, but they can't be nailed down explicitly, unless the answer lies in some book which I haven't consulted.

A secondary question is why ordinal numbers are used as "cardinality yardsticks" anyway. Why don't we use the infinite von Neumann universes as cardinality yardsticks instead? For example, Vω (which has the same cardinality as ω), Vω+1 which has the cardinality of the continuum, and Vω+ω which is reputedly the smallest model of Zermelo set theory. These spaces have the philosophical disadvantage that they don't include any "mediate cardinals". But they include every infinite cardinality which is of practical use in practical mathematics.

I'm not quite sure what you're asking for, but maybe this will satisfy you. You can give a reasonably explicit construction of a well-ordered set of order-type ω1 as follows. Let S be the set of all well-ordered sets (A,<) such that A is a subset of ℕ. Isomorphism of ordered sets gives an equivalence relation on S; let T=S/≅ be the set of equivalence classes. Order T by saying that [(A,<A)]<[(B,<B)] if there exists an isomorphism from (A,<A) to a proper initial segment of (B,<B) (this is clearly independent of the representatives of the equivalence classes chosen). Then it can be shown that < is a well-ordering on T, and (T,<) has order-type ω1.

More briefly, ω1 can be described explicitly as the set of possible lengths of well-orderings of countable sets. This construction also generalizes to higher cardinals: ω2 is the set of possible lengths of well-orderings of sets of cardinality ≤ℵ1, ω3 is the set of possible lengths of well-orderings of sets of cardinality ≤ℵ2, and so on.

@AlanU.Kennington Given your analysis background, you might like this: to each real number r I can associate a countable ordinal ω1r, the least ordinal which cannot be "computed" (in a precise sense, see https://en.wikipedia.org/wiki/Church%E2%80%93Kleene_ordinal) from r. Then each ω1r is a countable ordinal, and ω1 is the supremum of the ω1rs as r runs through the reals. This is no simpler than the "glue it all together" construction, in fact it's more complicated - but I'm making a wild guess that you might find this somewhat more satisfying. (cont'd)
Why? Because each real is "approximating" ω1 in a way, and the true ω1 is the limit of these approximations. But if you *do* find this at all satisfying, you should spend a bit to convince yourself that it is in fact exactly the same as the descriptions already given; I think then you'll be more satisfied with ω1 in general.

@AkivaWeinberger It's exactly the same, but *relativized to r*: ω1r is the least ordinal which does not have an r-computable copy, that is, the least ordinal α for which there is no binary relation R on ω which is r-computable and such that (ω,R) has order-type α. By a beautiful theorem of Sacks, these are exactly the *admissible ordinals* - those α such that there is no Σ1-definable-over-Lα surjective map from β to α for any β<α (think of this as saying that α is not obviously a non-cardinal).
@NoahSchweber I was asking about what "r-computable" means.
@AkivaWeinberger Oh, sorry. :P Saying "[thing] is r-computable" means there is a Turing machine, equipped with an *oracle* for r, which can compute [thing]. Basically, we give the Turing machine the additional ability to ask questions of the form "Is n in r?" for n a natural number. For instance, the complement of the Halting Problem is computable *relative to the Halting Problem*: on input n, just as "is n in the Halting Problem?" If "yes," output "0"; if "no," output "1". For formal details see e.g. http://www.isical.ac.in/~arijit/courses/spring2010/slides/complexitylec13.pdf.
(although note that there are many equivalent ways to define oracle machines, and that you should really look at a bunch of different definitions to see the general idea underneath them).
